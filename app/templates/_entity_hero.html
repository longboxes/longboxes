{# Shared hero/header macro for entity pages (volume, arc, issue). One macro, two variants. ``banner_url`` / ``dark_fill`` decide which: * Banner hero → ``banner_url`` (a CV screen image under a dark gradient) OR ``dark_fill`` (a plain dark panel, no art, for entities with no banner image — e.g. local volumes). Either way the title and meta float right-justified at the bottom. A ``cover_url`` is shown on the left as the hero's visual anchor when ``dark_fill`` is set, or when ``banner_cover`` opts a ``banner_url`` hero into the issue-style cover-on- banner layout. Used by volume + arc, the local volume page, and the character page. * Cover + text layout → neither given: cover thumb on the left, breadcrumb + title + meta on the right. Used by all three entity types when CV has no banner image for them. The issue page's banner branch is structurally distinct (cover thumb floats inside the banner alongside text) and stays inline in issue.html. Its no-banner branch uses this macro. The caller block receives three arguments so the same meta markup works in either variant: ``meta_class`` — primary meta text utilities. ``opacity-90`` on banners (light text on dark bg) or ``text-slate-600`` on the no-banner layout (dark text on light bg). ``subtle_class`` — muted divider/separator tone. ``opacity-60`` on banners, ``text-slate-400`` on no-banner. ``is_banner`` — boolean, True iff the macro picked the banner branch. The issue page passes it to send the cover into the reader. Works in both variants. ``cover_progress`` (optional) is a ProgressBar — a thin reading- progress bar is drawn along the cover's bottom edge.
